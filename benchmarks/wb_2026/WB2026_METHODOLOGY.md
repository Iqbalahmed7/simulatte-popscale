## 1. What This Is

This is a pre-election vote-share prediction for the West Bengal 2026 Legislative Assembly
election (294 seats), generated by Simulatte's population simulation engine. The engine
constructs synthetic personas representing Bengal's electorate, runs each persona through
a structured cognitive reasoning loop informed by electoral context, and aggregates their
voting decisions into a population-level forecast.

This is the first application of Simulatte's engine to a non-US state election.

---

## 2. Methodology

### 2.1 System Architecture

The Simulatte engine has two components:

**Persona Generator (PG):** Generates deep synthetic personas from demographic seeds.
Each persona is a structured record containing demographic anchors (age, gender, location,
religion, caste community), derived psychological insights (risk appetite, trust anchor,
decision style, value orientation), behavioural tendencies (price sensitivity, brand
loyalty), and a worldview anchor (political lean, religious salience, governance
attitudes). Generation uses Claude Haiku (claude-haiku-4-5-20251001).

**PopScale:** Runs personas through an electoral scenario. For each persona, it:
1. Renders a structured stimulus (the electoral question and context)
2. Appends domain-specific behavioural framing based on the persona's attributes
3. Calls the Persona Generator's cognitive loop: perceive → accumulate → decide
4. Extracts the voting decision from the output

The cognitive loop uses VOLUME tier (Claude Haiku throughout) for cost efficiency.

### 2.2 Population Pool

The Bengal population pool contains **98 synthetic personas** representing West Bengal's
diverse electorate. The pool was constructed with the following design principles:

**Demographic coverage:**
- 7 electoral sub-regions: Kolkata metropolitan, South Bengal rural, North Bengal
  (Koch-Rajbongshi), Jungle Mahal tribal belt, Matua belt (Nadia/N24Pgs), Muslim
  majority districts (Murshidabad/Malda), and urban middle class
- Community representation: Hindu general, Hindu OBC, Hindu SC, Hindu ST,
  Muslim general, Muslim OBC — proportional to Bengal's census composition
- Gender: ~43% women (reflecting women's political salience in Bengal)
- Age range: 23–68 (adult voting population)

**Political archetype distribution (declared prior to simulation):**

| Archetype | Count | Share | Rationale |
|---|---|---|---|
| TMC-lean (opposition + opposition_lean) | ~42 | ~43% | Muslim bloc + women welfare recipients + South Bengal rural |
| BJP-lean (bjp_supporter + bjp_lean) | ~34 | ~35% | Matua belt + North Bengal + Hindu consolidation + Jungle Mahal |
| Left-lean (left_lean) | ~9 | ~9% | CPM remnants + AIMIM-adjacent Muslims |
| Neutral | ~13 | ~13% | Genuine swing voters across all regions |

**Design note:** The pool archetypes were constructed using domain knowledge of Bengal's
2026 political landscape, informed by 2021 results, post-2021 BJP defections, and
community-level research. This is a declared assumption, not hidden tuning. The cognitive
loop then converts each persona's archetype + contextual reasoning into an actual vote
choice — the final vote shares emerge from that reasoning, not from the archetype
distribution directly.

### 2.3 Electoral Scenario

The scenario presents each persona with the following question:

> *"West Bengal is voting in its 2026 assembly election. Given everything you know about
> your community, your state, and your own circumstances — which party will you vote for?"*

Options presented:
1. TMC (Trinamool Congress — Mamata Banerjee)
2. BJP (Bharatiya Janata Party — Modi-backed state campaign)
3. Left-Congress alliance (CPI-M / INC joint front)
4. Other party / NOTA (regional, independent, or None of the Above)

The scenario context included the following real-world factors (all verified from
published sources before the election):
- TMC's 15-year incumbency and anti-corruption narrative (Sandeshkhali, cut money)
- TMC's welfare architecture: Lakshmir Bhandar, Swasthya Sathi, Duare Sarkar
- BJP's Hindu consolidation strategy + CAA/Matua mobilisation
- SIR voter-roll deletions: ~91 lakh names removed, concentrated in Muslim-majority
  and TMC-stronghold districts (Al Jazeera, 16 Apr 2026)
- AIMIM-AJUP alliance threatening to fragment Muslim vote in Murshidabad/Malda
- Left-Congress alliance as a third option for corruption/communalism-fatigued voters
- Direct Mamata Banerjee vs Suvendu Adhikari contest in Bhabanipur

### 2.4 Domain Framing

Each persona receives a domain-specific framing block appended to the scenario. For the
POLITICAL domain, this includes:
- Risk appetite translated to political risk tolerance
- Trust anchor translated to political information sources
- Value orientation translated to political priority
- **Explicit political lean sentence** — for India personas, the persona's archetype
  is injected as a direct statement of political identity (e.g., for `left_lean`:
  *"Your political identity is rooted in the Left-Congress alliance..."*).

This framing is generated by PopScale's `frame_persona_for_domain()` function and
appended to every decide() prompt.

### 2.5 Simulation Parameters

| Parameter | Value |
|---|---|
| Population size (n) | 80 personas (sampled from 98-persona pool) |
| Generation model | claude-haiku-4-5-20251001 |
| Simulation tier | VOLUME (all Haiku, perceive→accumulate→decide) |
| Concurrency | 20 simultaneous LLM calls |
| Timeout per persona | 90 seconds |
| Fallback on failure | Prior-based heuristic (political lean → vote) |
| Run date | 21 April 2026 |

## 3. What This Study Is Not

**It is not calibrated on WB 2021.** A proper out-of-sample calibration would require a
separately constructed 2021-era pool with 2021-appropriate archetype distributions. Using
the 2026 pool to retrodict 2021 would not be a valid test. This calibration is planned
for future election cycles.

**It is not a poll replacement.** This engine simulates deliberative reasoning, not
sampling of revealed preferences. It is more analogous to a structured focus group
than a telephone survey.

**The political archetype distribution is an input assumption.** The pool's lean
distribution (~43% TMC, ~35% BJP, ~9% Left, ~13% neutral) reflects the researchers'
prior beliefs about Bengal's electorate. The cognitive loop converts these priors into
vote choices — but cannot manufacture a BJP majority from a TMC-heavy pool. The declared
prior is the key epistemic commitment.

|---|---|---|---|
| **TMC** | **56.3%** | 43.0% | +13.3pp |
| **BJP** | **37.5%** | 41.0% | −3.5pp |
| **Left-Congress** | **6.3%** | 10.0% | −3.7pp |
| **Others** | **0.0%** | 6.0% | −6.0pp |
| **MAE vs polls** | | | **6.6pp** |

**Simulation parameters:** n=80 personas, run_id=20260420_200122, Haiku generation,
VOLUME tier simulation, 21 April 2026.

### Interpretation

The simulation predicts a **TMC majority** with a larger margin than current polls suggest.
The engine overestimates TMC vote share — a systematic bias that has been consistent
across all Bengal runs. The most likely causes:

1. **Muslim persona TMC loyalty is too strong in the cognitive loop.** The 2026 context
   (SIR voter roll deletions, AIMIM-AJUP fragmentation) creates conditions for some
   Muslim vote fragmentation that the VOLUME-tier (Haiku) reasoning does not fully model.

2. **Left-Congress underestimated.** The `left_lean` archetype fix (B-WB-5) moved
   Left-Congress from 0% to 6.3%, confirming the fix worked, but the CPM residual
   base (estimated ~10%) is still partially absorbed by TMC.

3. **Others = 0%.** AIMIM-AJUP alliance vote is not captured — the pool has no
   AIMIM-specific archetype for the Murshidabad/Malda corridor.

### Confidence range

With n=80 personas (±5.5pp binomial sampling error at 95% confidence):

| Party | Prediction | 95% CI |
|---|---|---|
| TMC | 56.3% | 44.8% – 67.8% |
| BJP | 37.5% | 26.0% – 49.0% |
| Left-Congress | 6.3% | 0.8% – 11.8% |
| Others | 0.0% | 0.0% – 4.5% |

**Directional call (locked):** TMC wins majority. BJP loses ground from 2021 (38.1%).
Left-Congress holds a residual vote share. TMC winning margin wider than polls indicate.
